## 项目概述

**项目名称**: EarningsPulse - 财报监察 APP

**项目描述**: 实时监控上市公司财报数据，通过数据挖掘分析业绩表现，检测到十分亮眼的业绩时自动提醒的多平台应用

---

## 核心功能

### 1. 实时财报数据采集（多数据源）
- SEC EDGAR API
- Yahoo Finance API
- Alpha Vantage API
- FinViz API
- Company PR feeds

### 2. 智能业绩分析算法
- 同比/环比增长
- 超预期检测（EPS beat rate）
- 利润率趋势
- 市场反应分析

### 3. 自定义阈值设置
- 营收增长率 > 50%
- EPS 超预期 > 20%
- 利润率提升 > 10%
- 连续 3 季正向指引

### 4. 多渠道提醒
- APP 推送
- Telegram Bot 通知
- 邮件摘要
- Web 浏览器通知

### 5. 数据可视化
- 交互式图表
- 趋势曲线
- 同业对比

### 6. 公司管理
- 追踪列表
- 收藏功能
- 分组管理

### 7. 财报日历
- 预约提醒
- 电话会议提醒
- 报告发布提醒

## 技术栈（原始规划）

### 后端
- **框架**: Python + FastAPI
- **数据库**: PostgreSQL + TimescaleDB（时序数据）
- **缓存**: Redis
- **任务队列**: Celery + Redis
- **数据湖**: AWS S3 + Athena（历史分析）
- **流处理**: Apache Kafka + Flink

### 前端
- **框架**: React Native + TypeScript
- **跨平台**: iOS + Android

### 基础设施
- **API 网关**: Nginx + JWT 认证
- **监控**: Prometheus + Grafana
- **日志**: ELK Stack
- **CI/CD**: GitHub Actions

**注意**: 后续技术栈已更新为 Next.js + Supabase 方案，详见 `tech-architecture/README.md`

## 风险与挑战

### 技术风险
- **数据一致性**: 多源数据同步挑战
- **实时性能**: 500+ 公司实时处理的可扩展性
- **API 限流**: 第三方 API 调用频率限制

### 合规风险
- **数据使用**: 各数据源的商用许可要求
- **投资建议**: 必须明确声明不提供投资建议
- **用户隐私**: GDPR、CCPA 合规要求

### 商业风险
- **市场竞争**: 现有产品（如 Seeking Alpha、FinViz）
- **用户获取**: 金融工具领域竞争激烈
- **变现策略**: 免费 + 高级订阅 vs 免费完整版
